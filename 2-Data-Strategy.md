# THEORY & ARCHITECTURE

Every document generated by Greymatter Docs needs structured data.

Instead of hardcoding values into Python, we'll store them in a database.

Our data layer follows a simple architecture:

```text
                  main.py
                     │
                     ▼
             CustomerRepository
                     │
                     ▼
              DatabaseManager
                     │
                     ▼
                  SQLite
```

Each layer has a single responsibility:

| Layer            | Responsibility                     |
| ---------------- | ---------------------------------- |
| SQLite           | Stores application data            |
| DatabaseManager  | Opens connections and executes SQL |
| Repository       | Reads and writes business data     |
| Main Application | Calls repositories                 |

This separation keeps SQL isolated from the rest of the application, making the code easier to maintain and test.

## Step 3 — Create the Database Manager

**database/database.py**

```python
import logging
import sqlite3

from config.settings import DATABASE_FILE

logger = logging.getLogger(__name__)


class DatabaseManager:
    """Handles SQLite connections."""

    def __init__(self):
        self.database_file = DATABASE_FILE

    def get_connection(self):
        """Create a SQLite connection."""
        try:
            connection = sqlite3.connect(self.database_file)
            connection.row_factory = sqlite3.Row
            logger.info("Database connection established.")
            return connection

        except sqlite3.Error as error:
            logger.exception("Unable to connect to database.")
            raise RuntimeError("Database connection failed.") from error
```

## Step 4 — Create the Database Schema

**database/schema.py**

```python
import logging

from database.database import DatabaseManager

logger = logging.getLogger(__name__)


def create_schema():
    """Create database tables."""

    database = DatabaseManager()

    create_customer_table = """
    CREATE TABLE IF NOT EXISTS customers (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        first_name TEXT NOT NULL,
        last_name TEXT NOT NULL,
        company TEXT,
        email TEXT NOT NULL,
        phone TEXT,
        address TEXT,
        city TEXT,
        country TEXT,
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );
    """

    try:
        with database.get_connection() as connection:
            cursor = connection.cursor()
            cursor.execute(create_customer_table)
            connection.commit()

            logger.info("Database schema created successfully.")

    except Exception as error:
        logger.exception("Schema creation failed.")
        raise RuntimeError("Unable to create database schema.") from error
```

    INSERT INTO customers (
        first_name,
        last_name,
        company,
        email,
        phone,
        address,
        city,
        country
    )
    VALUES (?, ?, ?, ?, ?, ?, ?, ?)

        with database.get_connection() as connection:
            cursor = connection.cursor()

            cursor.execute("DELETE FROM customers")

            cursor.executemany(insert_sql, customers)

        logger.exception("Database seed failed.")
        raise RuntimeError("Unable to seed database.") from error

## Step 6 — Create the Repository

**database/repositories.py**

```python
import logging

from database.database import DatabaseManager

logger = logging.getLogger(__name__)


class CustomerRepository:

    def __init__(self):
        self.database = DatabaseManager()

    def get_all_customers(self):
        """Return every customer."""

        query = """
        SELECT *
        FROM customers
        ORDER BY last_name
        """

        try:
            with self.database.get_connection() as connection:
                cursor = connection.cursor()

                cursor.execute(query)

                rows = cursor.fetchall()

                logger.info(
                    "Retrieved %s customers.",
                    len(rows)
                )

                return rows

        except Exception as error:
            logger.exception("Customer lookup failed.")
            raise RuntimeError(
                "Unable to retrieve customers."
            ) from error
```

## Step 7 — Update `main.py`

```python
from config.logger import logger

from database.repositories import CustomerRepository
from database.schema import create_schema
from database.seed import seed_database


def main():

    logger.info("=" * 60)
    logger.info("Greymatter Docs Starting")

    try:

        create_schema()

        seed_database()

        repository = CustomerRepository()

        customers = repository.get_all_customers()

        for customer in customers:
            logger.info(
                "%s %s | %s",
                customer["first_name"],
                customer["last_name"],
                customer["company"]
            )

    except Exception as error:
        logger.exception(error)


if __name__ == "__main__":
    main()
```

# CODE WALKTHROUGH

## `DatabaseManager`

This class is the only component that knows how to connect to SQLite.

By centralizing connection logic, changing the database engine in the future (for example, PostgreSQL) becomes much easier.

---

## `row_factory`

```python
connection.row_factory = sqlite3.Row
```

Instead of accessing columns by index:

```python
customer[1]
```

we can use descriptive names:

```python
customer["first_name"]
```

This improves readability and reduces errors.

---

## Context Managers

Notice the use of:

```python
with database.get_connection() as connection:
```

This ensures that the database connection is automatically closed, even if an exception occurs.

---

## Repository Pattern

Rather than scattering SQL throughout the application, all customer-related database operations are encapsulated within `CustomerRepository`.

Future repositories might include:

* `TemplateRepository`
* `DocumentRepository`
* `JobRepository`
* `AuditRepository`

Each repository focuses on a single business entity, promoting a clean and maintainable architecture.

---

## Transactions

Every write operation ends with:

```python
connection.commit()
```

SQLite keeps changes in a transaction until they are committed. If an exception occurs before the commit, the changes are rolled back, preserving database consistency.

# TROUBLESHOOTING

| Problem                                     | Cause                                          | Solution                                                                    |
| ------------------------------------------- | ---------------------------------------------- | --------------------------------------------------------------------------- |
| `sqlite3.OperationalError: no such table`   | Schema has not been created                    | Run `create_schema()` before querying the database.                         |
| Duplicate sample records                    | Seed script inserts data repeatedly            | Clear the table or use `DELETE FROM customers` before seeding.              |
| `database is locked`                        | Multiple processes are writing to the database | Ensure connections are closed promptly and avoid long-running transactions. |
| `TypeError: tuple indices must be integers` | `row_factory` is not configured                | Set `connection.row_factory = sqlite3.Row`.                                 |
| Changes are not saved                       | Missing `connection.commit()`                  | Commit every successful write transaction.                                  |
